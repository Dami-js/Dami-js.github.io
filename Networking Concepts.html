OSI Model

OSI provides a standard for different computer systems to be able to communicate with each other.
The OSI Model can be seen as a universal language for computer networking. It is based on the concept of splitting up a communication system into seven abstract layers, each one stacked upon the last.

Seven Layers of OSI Model

7. The application layer

The application layer serves as the interface between the user and the data. Software applications like web browsers and email clients depend on this layer to initiate and manage communication processes, enabling data transfer between the user and the underlying network.

Application layer protocols include HTTP as well as SMTP (Simple Mail Transfer Protocol is one of the protocols that enables email communications).

6. The Presentation Layer

The Presentation layer is responsible for formatting and structuring data to be effectively utilized by the application layer. It translates user inputs into a format that the application layer can process and understand, ensuring smooth interaction between the user interface and the underlying business logic

5. The Session Layer

The Session layer controls the start and end of communication between two devices. It keeps the connection open long enough to send all the data, then closes it to save resources. It also uses checkpoints to make sure data is transferred smoothly and can be recovered if something goes wrong.
